## 讲讲web worker 和 child_process.fork() 的区别

前者是浏览器的，后者是Node.js的
前者创建轻量级线程（与主线程共享页面资源但不共享执行栈，即多线程），
后者创建独立子进程（真正的多进程，运行一个新的Node实例）

## 讲讲各种模块导出方式的区别和语法

#### 1. ESM（静态，不可写在if里面）

（1）export 
	- 命名导出， 必须用解构方式导入
		```js
		// 
		export const a = 1;
		export function foo() {}
		export class Bar {}
		// 或者统一写
		const a = 1;
		function foo () {}
		export {a, foo};
		```

	- 默认导出
		```js
		export default function () {}
		// 或者
		const obj = { x: 1 };
		export default obj;
		```
		导入时，可以随意命名：
		`import anything from "./xx.js"`

#### 2. CJS（动态加载）
（1） module.exports（真正导出的对象）
	```js
	module.exports = {
    a: 1,
    foo() {}
	}
	// 导入
	const ins = require("./xx")
	```
（2）exports === module.exports
	but, 你只能往exports 里添加属性，不可重新赋值：
	```js
	exports.a = 1;
	exports.foo = ....
	// 但是以下行为错误❌
	exports = { a: 1 }; // ❌ 此时等于“断开了引用”
	```
	因为真正导出的仍是 module.exports。
